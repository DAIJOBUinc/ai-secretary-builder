# ClaudeCode_AI秘書_構築指示書 — 00 はじめに（全体の地図）

> **⚠️ 応答は必ず日本語で。** このガイドに従って作業する間も、完成後のAI秘書も、ユーザーへの応答・質問・確認・成果物はすべて日本語で書く（コマンド名・コード・固有名詞はそのまま）。英語では返答しない。

このガイドは AI秘書をゼロから作るための研修・持ち帰り教材。**3部構成**で、各部は1セッションで読み切れるサイズに分けてある。研修の回ごと・作業セッションごとに、**該当する1ファイルだけを Claude Code に読み込ませる**（こうするとAIのコンテキストが軽く、ヒアリングが丁寧になる）。

> **進む順**: 第1部（ガワ）→ 第2部（能力）→ 第3部（中身）。本質は **Logic（骨格・処理）を先に、Data（自社の事実）を後に**。最も時間がかかるコンテキスト充実を最後に回すことで、セットアップ中のコンテキスト圧迫を避ける。Step番号は通し番号のまま（並びを変えても各Stepの呼び名は不変）。

構築するものは全10ステップ。**Step 1〜6（コア）で毎朝動くAI秘書が完成**し、Step 7〜10（発展）は運用に慣れてから足す。各部の冒頭に「前提チェック」があるので、別セッションで開いても前の部の成果物が揃っているか自分で確認できる。

各Stepの最後に成果物を見せて確認を取り、**実際に動作確認してから**次へ進む（「できたはず」で進めない）。途中で詰まったら → [companion/setup-troubleshooting.md](companion/setup-troubleshooting.md)。

なお、**いきなり第1部から始める前に、下の「最短ルート」で30分動かして成功体験を作る**のがおすすめ。

## 最短ルート（まず30分で動かす）

いきなり10ステップは多い。**最初の30分で「動くAI秘書」を体験**してから、Step 1 以降で作り込む。非エンジニアでもここまでは必ず到達できる設計。

### 0. 環境診断（つまずきを先に潰す）

AI（Claude Code）はまず、ユーザーのPC環境を確認する：

- **OS**（Mac / Windows）— このガイドのコマンド例は Mac/Linux 前提。Windows は読み替え（`~` → `%USERPROFILE%`、シェルは PowerShell）。迷ったら **WSL** を入れると例がほぼそのまま通る
- **Claude Code が起動するか** — ターミナルで `claude` が立ち上がるか
- **必要ツール** — Git が入っているか、繋ぎたいコネクタ（カレンダー等）のアカウントがあるか

> 詰まったら → [companion/setup-troubleshooting.md](companion/setup-troubleshooting.md)（症状→原因→対処で引ける）

### 1. 事前準備チェック（研修受講者は開講前に）

- [ ] Claude Code をインストール
- [ ] Claude の有料プラン（Pro 等）に課金
- [ ] エディタ（VS Code 等）をインストール
- [ ] （任意）音声入力ツール・AI議事録ツール
- [ ] 「AIに任せたい業務 3〜5個」と、**機密を含まない**サンプル資料を手元に用意

### 2. 最小3手で「おはよう」を動かす

1. **カレンダーをコネクタで繋ぐ**（claude.ai → 設定 → コネクタ。＝Step 3 の最小版）
2. **置き場フォルダを作り、ごく短い `CLAUDE.md` を置く**（「私の予定を管理するAI秘書」程度の数行でよい。＝Step 1-2 の最小版）
3. **`/daily-schedule` を1個だけ作る**（Step 6 のテンプレをコピー）→ 「おはよう」と打つ

これで「朝の工程表が出る」体験ができる。**この成功体験を入口に、Step 1 から本格構築へ進む。**

> 最短ルートはあくまで"動く体験"。本番は **Step 1〜6（コア）** ＝フォルダ構造・CLAUDE.md・コンテキスト注入をきちんとやる工程。
