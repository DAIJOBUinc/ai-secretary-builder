# 人事・採用 向け スターターコンテキスト

> この役割の人がAI秘書を最速で戦力化するための初期セット。本体ガイド Step5（コンテキスト注入）から参照される。まずここから埋め、慣れたら広げる。
>
> 使い方：あなた（Claude Code）は、人事・採用担当であるユーザーに以下を順にヒアリングし、`context/` を下書きしていく。先に求人票・選考フロー図・就業規則・評価シートで埋められる所を埋め、足りない分だけ聞く。

## 1. まず入れる context/ ファイル（最初に用意する自社情報）

- `context/hiring/criteria.md` — 求める人物像・評価軸・各ポジションの必須/歓迎要件（合否判断のものさし）
- `context/hiring/process.md` — 選考フロー（書類→一次→二次→最終→オファー）と各段階の見るポイント
- `context/candidates/{候補者}.md` — 候補者ごとのカルテ。応募経路・経歴要約・各面接の評価・現在の選考段階（個人情報を含む。機密度高）
- `context/hr/policies.md` — 就業規則・人事制度（等級・給与制度・休暇・労務ルールの参照元）
- `context/hr/evaluation.md` — 評価基準（評価項目・等級定義・評価サイクルの運用）

> 補足：候補者カルテはフロー（各面接ログ）とストック（今の選考段階・総合評価）を分ける。就業規則・評価基準はLogic（規範）として上書きで最新を保ち、改定があったら `updated:` を直す。個人情報を含むので、Git管理する場合は `.gitignore` での扱いを Step10 で確認する。

## 2. 最初に渡すと効く自社資料（context/company/sources/ に置く）

- 求人票・募集要項（各ポジション） — スカウト文・面接設計の素材になる
- 評価シート・面接スコアリング基準 — 評価の一貫性を保ち、フィードバックを構造化できる
- 就業規則・人事制度（等級・給与・休暇・労務） — 労務問い合わせとルール参照の土台になる
- カルチャーデック・社員インタビュー記事 — 候補者への訴求メッセージを作る原料になる

## 3. 鉄板スキル（universal ＋ この役割の初回セット）

**universal（全員の土台・まず作る2つ）**
1. `/daily-schedule` — 朝、カレンダーとタスクから15分刻みの工程表を作る。面接・候補者対応・1on1・労務処理を現実的に組む
2. `/mtg-prep` — 面接・1on1前に相手の経歴・前段階の評価・確認すべき点を整理した準備資料を作る

**人事・採用の鉄板（universal に足す）**
3. `/interview` — 新ポジションの要件や評価軸を、着手前にAskUserQuestionで深掘りして spec に落とす（求人設計に活用）
4. `/meeting-email` — 日程調整・オファー連絡・お見送り連絡のメール下書きを作る（送信は必ず本人許可後）

**次に足す（優先順）**: 1on1・面接系スキル（`context/candidates/` を前提にカスタマイズ）→ 評価・労務FAQ系（評価シート下書き・就業規則の参照回答）

## 4. 人事・採用の Agent Team テンプレ — 候補者評価チーム

重要ポジションの見極めは、単発より**分担チーム**で評価の偏りを減らせる。`.claude/agents/` に定義し、スキルから起動する。

| エージェント | 役割 |
| --- | --- |
| 求人要件設計 | 募集ポジションの必須/歓迎要件・評価軸を構造化 |
| 経歴スクリーニング | 候補者の経歴を要件と照合し、確認点を抽出 |
| 面接設計 | 見極めるべき点に沿った質問・課題を設計 |
| 評価サマリ | 各面接の評価を統合し、合否の論点を整理 |
| 批判的レビュー | 評価の偏り・確証バイアス・見落としを指摘 |

> 候補者の個人情報を扱うので、チームに渡す情報も `.gitignore`・権限の扱いに注意（下の注意参照）。初回は単発の経歴整理から始めてよい（本体 第2部 Step 8 参照）。

## 5. この役割ならではの注意（Gotcha）

- **候補者・従業員の個人情報は機密**：学歴・現職・連絡先・選考状況・評価・給与・労務情報は最高機密。対外資料や社外送信物に候補者・従業員の実名・経歴を載せない。お見送り連絡の文面に他候補との比較や評価理由を書かない。メアド一致や同姓・同名だけで人物を同定せず、確証が無い箇所は「（要確認）」と明記する。外部に出る文面は `.claude/rules/external-materials.md` のチェックを通す。
- **評価・労務の最終判断は人間**：AIは下書き・論点整理・要件構造化まで。合否・等級・処遇・懲戒などの決定はAIに委ねない。評価サマリや就業規則の参照回答は「案」として扱い、確証の無い労務判断は社労士・専門家に確認する。
