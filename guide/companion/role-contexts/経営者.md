# 経営者 向け スターターコンテキスト

> この役割の人がAI秘書を最速で戦力化するための初期セット。本体ガイド Step5（コンテキスト注入）から参照される。まずここから埋め、慣れたら広げる。
>
> 使い方：あなた（Claude Code）は、経営者であるユーザーに以下を順にヒアリングし、`context/` を下書きしていく。白紙の質問票を渡すのではなく、先に手元資料（会社案内・事業計画・HP）で埋められる所を埋め、足りない分だけ聞く。

## 1. まず入れる context/ ファイル（最初に用意する自社情報）

- `context/company/overview.md` — 事業概要・主要サービス・収益モデル・誰に何を売って儲けているか
- `context/company/decisions.md` — 意思決定ログ。なぜその判断をしたかを日付つきで追記（上書きでなく追記運用）
- `context/owner/profile.md` — 経営者本人のプロフィール・大事にしている判断基準・優先順位・避けたいこと
- `strategy/{年度や四半期}.md` — 今期の目標・重点テーマ・撤退ラインなどの戦略コンテキスト
- `context/company/kpi.md` — 追っている主要指標と現状値（数字の機密度が高いので扱いは慎重に）

> 補足：経営者は「判断の文脈」が命。事実（context/）と判断の経緯（decisions.md）を分けておくと、後から「なぜこう決めたか」をAIが正しく踏まえて壁打ちできる。

## 2. 最初に渡すと効く自社資料（context/company/sources/ に置く）

- 事業計画書・中期計画 — 目標と前提が入っており、戦略の壁打ちの土台になる
- 直近の取締役会・経営会議の資料／議事録 — 今の論点と決定事項をAIが把握できる
- 月次の業績サマリー（PL・KPIダッシュボード等） — 数字を踏まえた相談ができる（機密度高、社外送信注意）
- 会社案内・コーポレートデック・HPのURL — 自社の建付けと対外メッセージを理解させる
- 過去の重要な意思決定メモ — 判断のクセ・価値観を学習させる原料になる

## 3. 鉄板スキル（universal ＋ この役割の初回セット）

**universal（全員の土台・まず作る2つ）**
1. `/daily-schedule` — 朝、カレンダーとタスクから15分刻みの工程表を作る。午前に重い判断業務、午後に会議・事務を寄せる
2. `/mtg-prep` — 取締役会・経営会議・面談の前に、論点・前回決定・関連資料を整理（「共有用」と「社内メモ」の2部構成）

**経営の鉄板（universal に足す）**
3. `/sparring` — 経営判断・戦略の壁打ち相手。賛成役ではなく、反論・別視点・見落とし・潰すべき不確実性を出させる
4. `/weekly-report` — 今週の成果・進捗・課題・来週の予定をまとめる。経営の定点観測に使う

**次に足す（優先順）**: `/clip`（投資・業界情報の取り込み）→ `/company-research`（投資・提携先の企業カルテ）→ `/interview`（新施策の着手前ヒアリング）

## 4. 経営の Agent Team テンプレ — 戦略分析チーム

判断の重い問い（新規事業・撤退・大型投資など）は、単発の壁打ちより**分担チーム**が深い。`.claude/agents/` に定義し、スキルから起動する。

| エージェント | 役割 |
| --- | --- |
| 問いの設計 | 分析すべき問いを構造化 |
| 市場調査 | 市場規模・トレンド・競合 |
| 定量分析 | 財務・KPI |
| 戦略設計 | 調査を統合し戦略オプションを設計 |
| 批判的レビュー | 弱点・リスクを指摘 |

> 初回は単発の調査エージェント1〜2個から。慣れたらこのチームを組む（本体 第2部 Step 8 参照）。

## 5. この役割ならではの注意（Gotcha）

- **壁打ちで「いいですね」を言わせない**：経営者の相談相手は同調すると価値ゼロになる。`/sparring` は必ず `context/company/decisions.md` と照合し、過去の判断と矛盾する点・反論・リスクを先に出させる。気持ちよく終わる相談は失敗だと心得る。
- **数字と戦略は最高機密として扱う**：売上・利益率・資金調達額・撤退ラインなどは `context/` には入れてよいが、対外資料・Slack・メールには絶対に出さない。経営者の口頭情報は社外秘の塊なので、外部送信物を作るときは `.claude/rules/external-materials.md` のチェックを必ず通す。
