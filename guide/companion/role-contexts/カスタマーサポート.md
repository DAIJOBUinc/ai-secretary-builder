# カスタマーサポート（CS・サポート） 向け スターターコンテキスト

> この役割の人がAI秘書を最速で戦力化するための初期セット。本体ガイド Step5（コンテキスト注入）から参照される。まずここから埋め、慣れたら広げる。
>
> 使い方：あなた（Claude Code）は、CS・サポート担当であるユーザーに以下を順にヒアリングし、`context/` を下書きしていく。先に製品マニュアル・FAQ・返信テンプレ集で埋められる所を埋め、足りない分だけ聞く。

## 1. まず入れる context/ ファイル（最初に用意する自社情報）

- `context/product/overview.md` — 製品・サービスの仕様・主要機能・対象ユーザー（回答の正確さの土台）
- `context/product/faq.md` — FAQ・既知の不具合・回避策（よくある問い合わせの一次回答源）
- `context/support/templates.md` — 返信テンプレ集（問い合わせ・お詫び・案内の定型文への索引）
- `context/support/escalation.md` — SLA・エスカレ基準（誰に・いつ上げるか、確約してよい範囲の線引き）

> 補足：FAQ・テンプレ・SLAはLogic（規範）なので上書きで最新を保つ。問い合わせの個別ログはフローとして残し、製品情報や仕様の最新値はストック（上書き）で正しい版だけを残す。

## 2. 最初に渡すと効く自社資料（context/company/sources/ に置く）

- 製品仕様・操作マニュアル — 回答の正確さと再現手順の根拠になる
- FAQ・過去の問い合わせと回答 — 一次回答の精度が上がり、定型化できる
- 返信テンプレ集 — お詫び・案内・回答の下書きが自社トーンで揃う
- SLA・エスカレ基準 — 確約してよい範囲と上げどころが明確になる
- 既知不具合・リリースノート — 「今起きている事象」を踏まえた案内ができる

## 3. 鉄板スキル（universal ＋ この役割の初回セット）

**universal（全員の土台・まず作る2つ）**
1. `/daily-schedule` — 朝、カレンダーとタスクから15分刻みの工程表を作る。問い合わせ対応・オンボーディング・定例を組む
2. `/mtg-prep` — オンボーディング面談・定例前に、相手の利用状況・過去のやり取り・関連資料を整理（「共有用」と「社内メモ」の2部構成）

**CS・サポートの鉄板（universal に足す）**
3. `/meeting-email` — 問い合わせ回答・お詫び・案内のメール下書きを作る（送信は必ず本人許可後）
4. `/clip` — FAQ・既知不具合・製品アップデート情報を要約・タグ付けして溜める（回答ナレッジの原料置き場）

**次に足す（優先順）**: FAQ回答ドラフト系（`context/product/` を前提に一次回答を起こす）→ 議事録要約系（オンボーディング/定例の記録）

## 4. CS・サポートの Agent Team テンプレ — 問い合わせ対応チーム

複雑・重要な問い合わせは、単発より**分担チーム**で抜けと誤回答を減らせる。`.claude/agents/` に定義し、スキルから起動する。

| エージェント | 役割 |
| --- | --- |
| 一次受け | 問い合わせ内容を整理し、不足情報を洗い出す |
| 分類 | 種別（不具合/仕様/要望/課金等）と緊急度を判定 |
| 回答ドラフト | FAQ・仕様を参照し回答案を作成 |
| エスカレ判断 | SLA・確約範囲を超えるかを判定し、上げ先を決める |
| レビュー | 確約しすぎ・事実誤りがないかを点検 |

> フロー：一次受け→分類→回答ドラフト→エスカレ判断→レビュー。初回は単発のFAQ回答・問い合わせ要約から。慣れたらこのチームを組む（本体 第2部 Step 8 参照）。

## 5. この役割ならではの注意（Gotcha）

- **約束できない範囲を断定回答しない**：仕様・SLA外の対応可否、納期、返金、将来の機能追加などは確約しない。不確実なものは「確認します」と止め、`context/support/escalation.md` の基準でエスカレする。FAQに無い事象を推測で断定すると、後で覆って信頼を失う。回答前に「これは確約してよい範囲か」を必ず確かめる。
- **問い合わせ本文の個人情報・顧客機密を残さない**：問い合わせには氏名・連絡先・契約情報・スクショ内のデータが混ざる。これを `context/` やテンプレ、社外送信物に不用意に残さない。事例化・テンプレ化するときは個人情報を伏せる。送信系（メール・チャット）は下書き→提示→許可後に送信の3ステップを守る。
