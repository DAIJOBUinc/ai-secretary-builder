# デザイナー 向け スターターコンテキスト

> この役割の人がAI秘書を最速で戦力化するための初期セット。本体ガイド Step5（コンテキスト注入）から参照される。まずここから埋め、慣れたら広げる。
>
> 使い方：あなた（Claude Code）は、デザイナーであるユーザーに以下を順にヒアリングし、`context/` を下書きしていく。先にブランドガイド・デザインシステム・過去の制作物・依頼ブリーフで埋められる所を埋め、足りない分だけ聞く。AI秘書が担うのは**デザインの周辺作業**であり、造形・ビジュアルそのものの制作ではない（下の注意参照）。

## 1. まず入れる context/ ファイル（最初に用意する自社情報）

- `context/design/system.md` — ブランド・デザインシステムガイド（カラー・タイポ・コンポーネント・余白ルール。生成する文章・指示書の前提になる）
- `context/design/assets.md` — アセット/素材の索引（ロゴ・アイコン・フォント・画像の所在とライセンス区分。どこに何があるかの台帳）
- `context/design/decisions.md` — 過去の制作物・デザイン判断の記録（なぜこの形にしたかの根拠。現行の決定はストック、検討の経緯はフロー）
- `context/design/briefs.md` — 案件ブリーフ・依頼一覧（誰から・何を・いつまでに・制約は何か）

> 補足：decisions は「フロー（制作ログ・検討の経緯）」と「ストック（今の正となる決定＝現行のデザインシステム）」を分ける。ログは追記、現行の決定は上書きで最新に保つ。古い決定がストックに残ると、AIが廃止したルールを前提に指示書を書いてしまう。改定があったら `updated:` を直す。

## 2. 最初に渡すと効く自社資料（context/company/sources/ に置く）

- ブランド・デザインシステムガイド — カラー・タイポ・コンポーネント・トーンの基準。指示書とFBの言語が揃う
- アセット/素材の索引（ロゴ・アイコン・フォント・画像） — どの素材が使えるか・ライセンス区分をAIが把握する土台になる
- 過去の制作物と、その時のデザイン判断メモ — 自社らしい意思決定をAIが学習し、ブリーフ作成の精度が上がる
- 制作依頼テンプレ・指示書フォーマット — ブリーフ／指示書ドラフトの型が定まり、抜け漏れが減る

## 3. 鉄板スキル（universal ＋ この役割の初回セット）

**universal（全員の土台・まず作る2つ）**
1. `/daily-schedule` — 朝、カレンダーとタスクから15分刻みの工程表を作る。制作ブロック・レビュー・関係者調整を現実的に組む
2. `/mtg-prep` — デザインレビュー・キックオフ前に、論点・前回のFB・関連資料を整理（「共有用」と「社内メモ」の2部構成）

**デザイナーの鉄板（universal に足す）**
3. `/clip` — 参考事例・UIトレンド・競合の表現を要約・タグ付けして溜める（インプットの原料置き場）
4. `/interview` — 制作依頼のゴール・要件・制約を着手前にAskUserQuestionで深掘りし、ブリーフ／指示書の spec に落とす（手戻り削減）

**次に足す（優先順）**: ブリーフ／指示書ドラフト系（依頼・FBを構造化された指示書に落とす）→ デザインクリティック系（案の論点出し・批判的レビュー）

## 4. デザイナーの Agent Team テンプレ — デザイン検討チーム

案の検討は、単発より**分担チーム**で論点の網羅性と客観性が上がる。`.claude/agents/` に定義し、スキルから起動する。AIが出すのはあくまで論点と整理であり、造形の良し悪しの最終判断は本人に残す。

| エージェント | 役割 |
| --- | --- |
| 参考リサーチ | テーマに沿った参考事例・UIトレンド・競合表現を収集 |
| 要件整理 | 依頼のゴール・制約・優先順位を構造化 |
| 論点出し | 案ごとに検討すべき観点・トレードオフを抽出 |
| 批判的レビュー | デザインクリティックの視点で弱点・誇張・一貫性の崩れを指摘 |
| まとめ | 各案の論点を統合し、判断材料を整理 |

> フロー：参考リサーチ→要件整理→論点出し→批判的レビュー（デザインクリティック）→まとめ。初回は単発の参考収集・要件整理から。慣れたらこのチームを組む（本体 第2部 Step 8 参照）。

## 5. この役割ならではの注意（Gotcha）

- **AIは造形そのものを作らない／最終検収は人間**：AI秘書が担うのはデザインの周辺作業（依頼・ブリーフの要件文章化、FBの整理・構造化、参考事例・トレンドのリサーチと蓄積、デザインシステム／アセットの台帳化・ドキュメント化、関係者調整・日程・議事録）まで。Figma等での造形・ビジュアル判断・最終検収は本人に残す。生成された指示書やFB整理は「案」として扱い、見た目の良し悪しと採否はAIに委ねない。
- **権利と機密に注意**：参考事例・素材・フォントには著作権／ライセンスがある。AIが集めた参考をそのまま流用せず、権利確認は必ず人間が行う（商用可否・帰属表記・改変条件を確認）。未公開のデザイン・クライアント案件のビジュアルは機密。外部・SNS・対外資料に出さない。外部に出る資料は `.claude/rules/external-materials.md` のチェックを通す。
