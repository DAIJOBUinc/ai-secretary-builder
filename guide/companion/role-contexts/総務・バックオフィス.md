# 総務・バックオフィス 向け スターターコンテキスト

> この役割の人がAI秘書を最速で戦力化するための初期セット。本体ガイド Step5（コンテキスト注入）から参照される。まずここから埋め、慣れたら広げる。
>
> 使い方：あなた（Claude Code）は、総務・バックオフィス（総務・法務・庶務・規程・社内ナレッジ）であるユーザーに以下を順にヒアリングし、`context/` を下書きしていく。先に社内規程・契約ひな型・備品台帳・取引先一覧で埋められる所を埋め、足りない分だけ聞く。

## 1. まず入れる context/ ファイル（最初に用意する自社情報）

- `context/company/rules.md` — 社内規程・運用マニュアル（庶務・申請・社内フロー・各種ルール）
- `context/company/contracts.md` — 契約ひな型への索引（NDA・業務委託・利用規約などの定型雛形）
- `context/company/assets.md` — 備品・資産台帳（端末・ライセンス・設備の管理状況）
- `context/company/vendors.md` — 取引先一覧（庶務・サービス契約まわりの窓口・契約状況）

> 補足：規程・ひな型・台帳はストック（上書き運用）として「今正しい版」を一つに保ち、改定があったら `updated:` を直す。問い合わせや依頼の個別ログはフローとして残す。

## 2. 最初に渡すと効く自社資料（context/company/sources/ に置く）

- 社内規程・運用マニュアル — 問い合わせ対応とルール参照の土台になる
- 契約書の定型フォーマット（NDA・業務委託等） — 契約ひな型ドラフトの素材になる
- 備品・資産台帳 — 貸与・棚卸し・更新管理の整理に効く
- 取引先一覧（窓口・契約状況付き） — 庶務・契約まわりの抜け漏れ防止に効く

## 3. 鉄板スキル（universal ＋ この役割の初回セット）

**universal（全員の土台・まず作る2つ）**
1. `/daily-schedule` — 朝、カレンダーとタスクから15分刻みの工程表を作る。申請処理・社内問い合わせ・庶務を漏らさず組む
2. `/mtg-prep` — 社内定例・取引先MTG前に、関連する規程・前回決定事項・関連資料を集めて準備資料を作る

**総務・バックオフィスの鉄板（universal に足す）**
3. `/clip` — 法改正・制度変更・新しい業務ツールの記事を要約・タグ付けして溜める（社内ナレッジの原料置き場）
4. `/slack-digest` — 社内からの問い合わせ・依頼をトリアージして、対応漏れを防ぐ

**次に足す（優先順）**: 定型文書ドラフト系（規程・社内通知・契約ひな型を自社雛形で）→ 議事録要約系

## 4. 総務・バックオフィスの Agent Team テンプレ — ナレッジ整備チーム

規程改定・制度対応など「調べて・整理して・文書化する」重い仕事は、**分担チーム**が速い。`.claude/agents/` に定義し、スキルから起動する。

| エージェント | 役割 |
| --- | --- |
| 情報収集 | 法改正・制度変更・社内ルールの一次情報を収集 |
| 分類整理 | 集めた情報を規程・FAQ・テンプレの体系に分類 |
| ドキュメント化 | 自社フォーマットで規程・通知・FAQドラフトを作成 |
| 整合チェック | 既存ルールとの矛盾・古い版の混在を点検 |

> 初回は単発の情報収集・要約から。慣れたらこのチームを組む（本体 第2部 Step 8 参照）。契約・個人情報の扱いに注意（下の注意参照）。

## 5. この役割ならではの注意（Gotcha）

- **契約・規程の最終確認は人間/専門家**：契約ひな型・規程・通知のドラフトをAIに出させても鵜呑みにしない。法的効力・労務リスクのある条文は、法務・社労士・弁護士のチェックを必ず通す。AIの生成物は「たたき台」として扱い、確証の無い解釈をそのまま社内外に流さない。送信系（メール・Slack）は下書き→提示→許可後に送信の3ステップを守る。
- **経理業務は「経理・財務」役割へ**：請求・支払・記帳・勘定科目・仕訳は、このテンプレの対象外（→ `経理・財務.md`）。ここは契約・庶務・規程・社内ナレッジに集中する。金額や口座が絡む依頼が来たら、経理・財務の context/ とルールに沿って扱う。
