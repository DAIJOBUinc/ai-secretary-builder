# 営業 向け スターターコンテキスト

> この役割の人がAI秘書を最速で戦力化するための初期セット。本体ガイド Step5（コンテキスト注入）から参照される。まずここから埋め、慣れたら広げる。
>
> 使い方：あなた（Claude Code）は、営業であるユーザーに以下を順にヒアリングし、`context/` を下書きしていく。先に手元の提案資料・価格表・既存顧客リストで埋められる所を埋め、足りない分だけ聞く。

## 1. まず入れる context/ ファイル（最初に用意する自社情報）

- `context/company/overview.md` — 自社が何を売っているか・誰に効くか・選ばれる理由（提案トークの土台）
- `context/company/pricing.md` — 料金体系・単価・割引条件（提案・見積の根拠。機密度高、社外向けは要確認）
- `context/clients/{顧客}.md` — 主要顧客ごとのカルテ。担当者・課題・商談履歴・温度感・次アクション
- `context/clients/index.md` — 顧客一覧と「対外資料に載せてよいか」の許諾メモ（事例公開の可否を管理）
- `context/company/sales_playbook.md` — 提案の型・よくある反論への切り返し・受注パターン

> 補足：顧客カルテは「フロー（商談ログ）」と「ストック（今の状態・次アクション）」を分ける。ログは追記、今の状態は上書きで最新に保つ。

## 2. 最初に渡すと効く自社資料（context/company/sources/ に置く）

- 標準提案デック・サービス紹介資料 — 提案ドラフトの素材になる
- 価格表・見積テンプレート — 金額の根拠を踏まえた提案・フォローができる
- 既存顧客リスト（社名・業種・担当・ステータス） — 顧客カルテの初期データになる
- 過去の受注/失注事例メモ — 勝ち筋・負け筋をAIが学習し、提案の精度が上がる
- 商談議事録（AI議事録ツールの書き出し等） — 次アクションとフォロー文面の精度が上がる

## 3. 鉄板スキル（universal ＋ この役割の初回セット）

**universal（全員の土台・まず作る2つ）**
1. `/daily-schedule` — 朝、カレンダーとタスクから15分刻みの工程表を作る。商談・移動・準備を現実的に組む
2. `/mtg-prep` — 商談前に相手企業・過去のやり取り・関連資料を集め、「共有用」と「社内メモ（金額・方針）」の2部構成で準備資料を作る

**営業の鉄板（universal に足す）**
3. `/meeting-email` — 商談の事前リマインド／事後フォローのメール下書きを作る（送信は必ず本人許可後）
4. `/company-research` — 新規リードや初回商談の相手企業を深掘りした「企業カルテ」を生成する

**次に足す（優先順）**: 商談フィードバック／ナーチャ漏れ通知系（自社のCRM・案件管理に合わせて）→ `/clip`（業界・競合情報の取り込み）

## 4. 営業の Agent Team テンプレ — 提案書作成チーム

大型・重要提案は、単発より**分担チーム**で深さと説得力が出る。`.claude/agents/` に定義し、スキルから起動する。

| エージェント | 役割 |
| --- | --- |
| 顧客リサーチ | 相手企業の課題・ニュース・業界動向 |
| 競合分析 | 競合の提案・価格帯・差別化 |
| ソリューション設計 | 顧客課題への解決策 |
| コピーライター | 説得力のある提案本文 |
| レビュアー | 顧客視点で穴・弱点を指摘 |

> 初回は単発の企業リサーチ・議事録要約から。慣れたらこのチームを組む（本体 第2部 Step 8 参照）。提案書に載せる顧客名・金額・他社事例は許諾を確認（下の注意参照）。

## 5. この役割ならではの注意（Gotcha）

- **mtg-prep の「共有用」に社内情報を混ぜない**：準備資料は2部構成。値引き上限・原価・相手担当者の人物評価・社内の交渉方針は必ず「社内メモ」側に置き、相手に見せる「共有セクション」に絶対に出さない。1ファイルで両方を作るので、誤って共有側に貼る事故が起きやすい。送る前に共有セクションだけを見直す。
- **顧客名・金額・他社事例の流用に注意**：提案書や事例に他顧客の社名・契約金額・担当者実名を載せるのは、`context/clients/index.md` の許諾を確認してから。許諾が無ければ「大手製造業A社」等に抽象化する。外部に出る資料は `.claude/rules/external-materials.md` のチェックを通す。
