# セットアップで詰まったとき（Troubleshooting / Gotchas）

本体ガイドの各ステップから参照される「詰まりどころ集」。**症状 → 原因 → 対処**で引けるようにしてある。

> このファイルの育て方：ここに載っているのは「**実際に詰まると分かっている**」項目だけ。研修・運用で新しく詰まった所が出たら、**机上で想像せず、実際に起きたものだけ**を末尾の「研修で見つかった詰まりどころ」に追記していく（[改善ループ](#改善ループ)参照）。

---

## よくある詰まりどころ

### 1. コネクタが繋がらない／一覧に出ない
- **症状**: 「設定 → コネクタ」を開いても繋ぎたいツールが無い／繋いだのにAIから使えない。
- **原因**: コネクタ接続は **Claudeアプリ（claude.ai / デスクトップ）のブラウザGUI操作**。Claude Code（ターミナル）側からは繋げない。Gmail・Googleカレンダー等の一部は claude.ai 側での接続が必須。
- **対処**: ブラウザで claude.ai → 設定 → コネクタ から接続 → Claude Code を再起動 → 「カレンダーの今日の予定を出して」等で動作確認。一覧に無いツールは CLI/MCP（本体 Step3 の Tier2/3）へ。

### 2. Mac と Windows でコマンドが違う
- **症状**: ガイドのコマンド例（`~/...`、`bash`、`ls` 等）がそのまま動かない。
- **原因**: 本体の例は Mac/Linux 前提。Windows はパス区切り（`\`）・シェル（PowerShell）・ホーム表記が異なる。
- **対処**: Windows では `~` → `%USERPROFILE%`、`/` → 環境に応じて読み替え。WSL を使うと Mac/Linux の例がほぼそのまま通る（迷ったら WSL 推奨）。

### 3. 毎回「許可しますか？」が出て進まない
- **症状**: ファイル読み書きやコマンド実行のたびに確認が出る。
- **原因**: よく使うツールが事前許可されていない。
- **対処**: `.claude/settings.json` の `allow` に頻用ツールを追加（例 `"Bash(gh:*)"` `"Read"` `"Edit"`）。本体 Step4 のサンプル参照。`deny` は壊す系・機密読取だけに絞る。

### 4. MCP が繋がらない
- **症状**: MCPツール（VOICEVOX・画像生成等）が呼べない。
- **原因**: 登録漏れ／URL・起動コマンド誤り／Claude Code 未再起動。
- **対処**: `claude mcp add` で登録したか確認 → `claude mcp list` で状態確認 → Claude Code 再起動。ローカル系（VOICEVOX等）はエンジン本体が起動しているかも確認。

### 5. 音声（VOICEVOX 等）が鳴らない
- **症状**: 音声フィードバックが出ない／エラーが出る。
- **原因**: 音声エンジンが起動していない。
- **対処**: **音声は任意機能。鳴らなくても構築は進められる**ので、まずは無視して先へ。鳴らしたい場合のみエンジンを起動。

### 6. スラッシュコマンド（スキル）が出てこない
- **症状**: `/daily-schedule` 等を打っても認識されない。
- **原因**: ファイルが `.claude/commands/` に無い／ファイル名とコマンド名が不一致／再起動前。
- **対処**: `.claude/commands/{名前}.md` があるか・frontmatter（model/description）が壊れていないか確認 → Claude Code 再起動。

### 7.「おはよう」で工程表が出ない
- **症状**: トリガーワードを打っても `/daily-schedule` が動かない。
- **原因**: カレンダー未接続／CLAUDE.md の「ワークフロー定義」にトリガーが書かれていない。
- **対処**: カレンダー接続を確認（詰まり#1）→ CLAUDE.md に「『おはよう』→ `/daily-schedule`」を記載 → スキル本体が存在するか確認（詰まり#6）。

### 8. 会社資料（PDF/Office）が読み込めない
- **症状**: `context/company/sources/` に置いた資料の中身を AI が拾えない。
- **原因**: 対応スキル（pdf/docx/xlsx/pptx）が無い／ファイルが破損・画像PDF（スキャン）。
- **対処**: 対応スキルを使う、または中身をテキストで貼ってもらう。スキャンPDFは文字起こしが要る場合あり。HP は URL を渡せば `WebFetch` で取得できる。

## 研修で見つかった詰まりどころ（追記していく）

> ここは **実際の研修・運用で詰まった事例だけ** を追記する欄。日付＋症状＋対処の形で。机上の想像は書かない。

### 2026-06-19 構築指示書を読み込ませるとAIが英語で返答する
- **症状**: 構築指示書（00〜第3部）を Claude Code に投げると、応答・確認・成果物が英語で返ってくることがある。
- **原因**: 明示的な言語指定が無いと、AIが英語で応答することがある（英語まじりの技術用語・コマンドが多いセットアップ中は特に）。
- **対処**: 各ガイドファイル冒頭の「**応答は必ず日本語で**」に従う。英語で返ってきたら「日本語で答えて」と一言指示する。完成後のAI秘書は CLAUDE.md 絶対ルール#1「言語」で恒久的に日本語固定。

## 改善ループ

このファイルは「回を重ねるごとに強くなる」設計：

1. 研修・運用で**実際に詰まった所**を拾う（受講者の質問・サポートログ）
2. 上の「研修で見つかった詰まりどころ」に追記（症状→原因→対処）
3. 頻出するものは本体ガイドの該当ステップにも `⚠️` リンクを足す
4. リポに issue/PR で持ち寄れば、メンバー全員の発見が溜まる

机上で詰まりを創作しない。**本物の失敗だけが良い Gotcha になる。**
